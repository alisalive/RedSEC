# RedSEC — Claude Code Notes

## Project overview

RedSEC is a red team log aggregation and correlation engine.  It parses tool
output (nmap, nuclei, ffuf, hydra, …), normalises events into a shared schema,
correlates them into attack chains, and exports SEC-compatible rule files.

## Correlation rule types

Rules live in `redsec/correlation/rules/default.yaml`.  The engine
(`redsec/correlation/engine.py`) supports four rule types controlled by the
`type:` YAML field (default: `sequence`).

| `type` value       | Dataclass        | Behaviour |
|--------------------|------------------|-----------|
| `sequence`         | `_Rule`          | Ordered event sequence within a time window. Requires `conditions` list (≥ 2 entries). |
| `pair_with_window` | `_PairRule`      | Watches for a `first` event then a `second` within `window_seconds`. Produces an `on_match` chain or an `on_timeout` chain. |
| `context`          | `_ContextRule`   | Captures a context field (`target`, `tool`, or `port`) from a `trigger` event, then compares it against the same field on the next `match` event in the window. Produces an `on_match` chain (same value) or `on_miss` chain (different value). No chain if no match event found. |
| `synthetic`        | `_SyntheticRule` | Counts `trigger` events; when `threshold` is reached within `window_seconds` a synthetic `RedSecEvent` (tool=`redsec`) is generated and all trigger events + synthetic are grouped into an `is_synthetic=True` chain. |

## Key model fields (AttackChain)

- `pair_type` — `"pair_with_window"` or `"context"` for those rule types; `None` for sequence chains.
- `pair_on_match`, `pair_on_timeout` — outcome messages; for context chains `pair_on_timeout` stores the `on_miss` message.
- `pair_window_seconds`, `pair_second_type` — window and second event type for pair/context chains.
- `is_synthetic` — `True` for chains produced by synthetic rules.

## SEC export (`redsec/exporters/sec.py`)

| Chain type         | SEC rules generated |
|--------------------|---------------------|
| sequence           | One `type=Single` per event + one `type=SingleWithThreshold` completion rule |
| `pair_with_window` | One `type=PairWithWindow` rule via `_pair_rule()` |
| `context`          | One `type=PairWithWindow` rule via `_context_rule()` (pattern2 uses trigger's target to enforce same-context matching) |
| synthetic          | `# SYNTHETIC EVENT CHAIN` header + one `type=Single` per event + `type=SingleWithThreshold` |

## ToolName

`redsec.redsec` was added to `ToolName` for synthetic events generated by the
correlation engine.

## Tests

- `tests/test_correlation.py` — sequence, pair, context, and synthetic rule tests.

- `tests/test_sec_exporter.py` — SEC exporter tests.

Run all tests: `pytest tests/ -v`
